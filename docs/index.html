Import data of any shape
into a relational database.

Loadstone turns nested XML, JSON, and curated CSV files into clean relational rows — driven by one declarative manifest per dataset, with durable queues, pluggable reference-data lookups, and row-level error forensics built in.

One manifest replaces every hand-written importer

Declare the shape once; Loadstone derives everything else.

Any format in, tables out

XML, JSON, and CSV (including zip-of-CSVs hierarchies) stream through one canonical record model. Same manifest, same tables, three formats — files of any size, bounded memory.

Hierarchies are first-class

Parents with nested, optional, repeating children to any depth. Bulk staging and set-based merges wire the foreign keys automatically — no ORM, no per-row round trips.

Bad rows don't kill good files

Records that fail validation or lookup resolution are quarantined with the source line, field, raw value, and reason. The rest of the file imports. Re-runs are idempotent.

Bring your own reference data

Built-in code lists, configuration-defined SQL lookups against any existing database, or a custom provider class. Per-field policies: reject record, reject file, default, auto-create.

Durable queues, no broker

Every dataset gets a named, database-backed queue: restart-safe, multi-worker, exponential-backoff retries, dead-lettering. Nothing extra to install or operate.

Observable end to end

A live dashboard, structured logs, OpenTelemetry traces and metrics, correlation ids, job timelines, and queryable rejection reports — over HTTP, not buried in log files.

Declare it

Entities, types, keys, lookups — one JSON document.

{
  "name": "orders",
  "root": {
    "name": "Order",
    "table": "Orders",
    "keyColumn": "OrderId",
    "naturalKey": [ "OrderNumber" ],
    "fields": [
      { "name": "OrderNumber", "required": true },
      { "name": "OrderDate", "type": "date" },
      { "name": "Country",
        "lookup": { "list": "countries",
                    "onMissing": "autoCreate" } }
    ],
    "children": [ { "name": "Line", "table": "OrderLines",
                    "parentKeyColumn": "OrderId",
                    "naturalKey": [ "LineNumber" ],
                    "fields": [ /* ... */ ] } ]
  }
}

Run it

API-first; everything scriptable, everything visible.

# register (or update) the dataset
curl -X PUT :8080/api/datasets/orders \
     --data-binary @orders.dataset.json

# generated DDL, if you want Loadstone
# to create the target tables
curl :8080/api/datasets/orders/schema

# queue an import — xml, json, or csv/zip
curl -X POST :8080/api/datasets/orders/imports \
     -F "file=@orders.xml"

# watch it: status, timeline, rejections
curl :8080/api/imports/{jobId}
curl :8080/api/imports/{jobId}/events
curl :8080/api/imports/{jobId}/rejections
